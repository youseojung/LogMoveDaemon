[IIS_LOG_MANAGE 개발 사항]
1. .NET 40으로 개발 (SUNSHINE 서버 .NET40으로 되어 있음)

2. 기존 APP.CONFIG 방식에서 외부 XML 형식으로 변경
	2-1. 동적 배열 구성으로 변경.
	2-2. SRC와 DEST의 구성 수는 일치해야 함.

3. SEVENZIP 사용시 SEVENZIP프로그램이 설치되어 있어야 함.
	3-1. 64BIT OS일 경우 64BIT버전 설치해야 함.

4. 기존 소스에서 LIBRARY 형식으로 되어 있는 부분 제거.
	4-1. FILEDIR만 가져옴.

5. WINDOWS DOMAIN NETWORK 공유 기능을 사용하여 운영됨.
   따라서 작업시 고급공유 기능을 사용하여야 함.
   순서는 고급공유 -> 권한 -> 공유사용권한 추가 -> entbackup 입력 -> 이름확인 및 확인 -> 권한 전체 선택 -> 확인.

6. 스케줄 등록시 entbackup 계정으로 스케줄 등록함.
